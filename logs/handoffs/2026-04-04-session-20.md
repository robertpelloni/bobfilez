## Key technical findings
1. The repo could not complete a practical default Windows build while native FFmpeg compilation was forced through vcpkg on this host.
2. The in-process video/audio providers are already feature-guarded in code, so making FFmpeg/Chromaprint opt-in was a safe stabilization move.
3. Full GUI/Omni validation is still environment-blocked by missing Qt6 package discovery, not by the headless build anymore.
4. Once the dependency wall was reduced, several real source issues surfaced across the recently expanded subsystem interfaces and implementations; these are now fixed for the headless profile.

## Remaining gaps
- Full `FO_BUILD_GUI=ON` / `FO_BUILD_OMNI=ON` validation still requires Qt6 on the machine.
- The optional `media-analysis` feature was not revalidated in this session after being moved behind a vcpkg feature gate.
- Many Omni-era engines remain architectural scaffolds rather than production-complete backends.

## Recommended next step
Install/configure Qt6 and resume a full GUI/Omni build, using `scripts/build_headless.bat` as the stable fallback verification path whenever the shell environment is unavailable.
