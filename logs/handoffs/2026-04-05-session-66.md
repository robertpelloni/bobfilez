## Important Findings
- The BobGUI integration path is now no longer binary between "CLI bridge only" and "full direct C++ integration".
- There is now a practical middle layer:
  - **direct C ABI**
  - small enough for C-native consumers
  - broad enough to expose real workflows
  - narrow enough to avoid freezing a huge C struct ABI too early
- Returning JSON through the C API is a good transitional design because it keeps the ABI small while staying friendly to GUI consumers.
