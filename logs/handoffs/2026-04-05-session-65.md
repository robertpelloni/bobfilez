## Important Findings
- The BobGUI lane needed progress, but a direct BobGUI-to-`fo_core` link is still a larger infrastructure task than the other frontend lanes because it crosses:
  - Meson ↔ CMake
  - C ↔ C++20
  - GLib/BobGUI ↔ bobfilez core runtime/toolchain boundaries
- The CLI bridge is therefore the correct immediate step because it is:
  - functional
  - CLI-first
  - honest about host limitations
  - a strong stepping stone toward a later C ABI shim
