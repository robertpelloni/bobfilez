## Main findings
### Current shell/UI size
- 49 QML files
- 9,844 QML lines
- 39 route-driven shell/panel surfaces

### Most important architectural finding
Current BobUI widgets/layouts are themselves built on:
- `QQuickItem`
- `QQuickPaintedItem`

So a BobUI-first migration can reduce or replace stock Qt modules, but it does **not** currently support a true no-`QtQuick` end state.

### Practical migration ranking
1. Remove `WebEngineQuick` → feasible first step
2. Reduce `QtCharts` / `GraphicalEffects` / stock controls → harder, but incremental
3. Remove most QML composition → major rewrite
4. Remove `QtQuick` entirely → not realistic with current BobUI architecture

### Additional integration finding
Current bobfilez bootstrap still registers only:
- `FileModel`
- `TreemapModel`

It does not yet register the broader BobUI QML type surface, so the current shell is still more stock Qt Quick than true BobUI-native UI.

## Recommended next step
Start with the smallest high-value cut:
- replace `MarkdownViewerPanel.qml` / `QtWebEngine`

That reduces one of the current hard BobUI compatibility blockers without forcing a full frontend rewrite.
